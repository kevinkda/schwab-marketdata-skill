# Quick Start — Step 3: Run your first OAuth flow

> **Goal**: Complete one end-to-end Schwab 3-legged OAuth dance and
> persist `token.json` to `~/.local/state/schwab-marketdata-mcp/`.

## Prerequisites

- [ ] Step 2 completed and `.env` has all three fields populated
- [ ] Your browser can reach <https://127.0.0.1:8182> (`login_flow`)
      or you can manually copy the callback URL (`manual_flow`)

## Which mode to use

| Scenario | Recommended |
| -------- | ----------- |
| Regular laptop / can listen on local https port 8182 | `login_flow` (default) |
| SSH-only / remote container / WSL2 / no browser access to `127.0.0.1` | `manual_flow` |
| Cron / unattended automation | **Not supported** — Schwab OAuth always requires a human one-time |

For the full decision tree, see [`../oauth/oauth-overview.md`](../oauth/oauth-overview.md).

## A. `login_flow` (default)

```bash
cd /path/to/schwab-marketdata-mcp
uv run python -m schwab_marketdata_mcp.auth login_flow
```

Behavior:

1. The CLI starts a local https listener (port `8182`, self-signed cert).
2. It opens the browser automatically to the Schwab login page.
3. You log in and click Allow → Schwab redirects to
   `https://127.0.0.1:8182/...`.
4. The browser pops one **ERR_SSL_PROTOCOL_ERROR / Self-signed cert**
   warning: click **Advanced → Proceed** (the cert is auto-generated by
   schwab-py and legitimate).
5. The CLI reads the callback URL → exchanges with App Secret for
   access_token + refresh_token → writes to
   `~/.local/state/schwab-marketdata-mcp/token.json` (`chmod 600`).

## B. `manual_flow` (headless)

```bash
uv run python -m schwab_marketdata_mcp.auth manual_flow
```

Behavior:

1. The CLI **prints an OAuth URL to stdout** (something like
   `https://api.schwabapi.com/v1/oauth/authorize?...`).
2. You **copy** that URL, paste it into any browser that can reach
   Schwab, log in, click Allow.
3. Schwab redirects to `https://127.0.0.1/?code=...&state=...` — the
   browser shows `Connection refused` — **this is expected**.
4. **Fully copy** the redirect URL from the address bar (including the
   entire `?code=...` query string) and paste it back into the CLI.
5. The CLI uses that URL to complete the token exchange.

## Expected outcome

- `~/.local/state/schwab-marketdata-mcp/token.json` exists
- mode = `600`, parent directory mode = `700`
- File content is valid JSON containing `access_token` /
  `refresh_token` / metadata fields

## Verification checklist

- [ ] `ls -l ~/.local/state/schwab-marketdata-mcp/token.json`
      → `-rw------- ... token.json`
- [ ] `cat ~/.local/state/schwab-marketdata-mcp/token.json | python -m json.tool`
      exits 0
- [ ] `stat -c '%a' ~/.local/state/schwab-marketdata-mcp` = `700`
      (macOS: `stat -f '%A'`)
- [ ] CLI stdout contains `Token persisted to ...token.json`
      (the standard log line in schwab-py 1.5+)

## Common failures

| Symptom | Fix |
| ------- | --- |
| Browser `ERR_SSL_PROTOCOL_ERROR` | Click Advanced → Proceed; the self-signed cert is expected |
| `MismatchingStateException ("CSRF Warning!")` | `.env`'s `SCHWAB_CALLBACK_URL` does not match the Portal's registered value; fix it; see [`../oauth/oauth-callback-mismatch.md`](../oauth/oauth-callback-mismatch.md) and [`../troubleshooting/auth-callback-url-mismatch.md`](../troubleshooting/auth-callback-url-mismatch.md) |
| Port 8182 already in use | `lsof -i :8182` to find the process; stop it or pick another port (and update the Portal's registered value) |
| In `manual_flow`, the callback URL has no `?code=...` | Schwab denied the authorization; usually the App did not actually enable the Market Data Production product, go back to [Step 1](step-1-developer-portal-app.md) and check |
| token landed in iCloud / Dropbox sync folder | `SchwabAuthError(reason="cloud_path_detected")`; use `--config-dir ~/.local/state/schwab-marketdata-mcp` explicitly, or `--i-understand-cloud-sync-risk` as last resort |

## What not to do

- **Do not** paste `token.json` content into chat / issues / PRs.
- **Do not** copy token.json to another machine (refresh_token is
  IP-bound + rotate-on-use; the old machine's copy is invalidated
  immediately).
