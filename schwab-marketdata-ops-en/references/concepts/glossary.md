# Glossary

> Cross-cutting glossary spanning the skill / MCP server / Schwab API.
> Grouped by topic.

## OAuth & authentication

| Term | Definition |
| ---- | ---------- |
| **App Key** | The ~32-character client identifier issued when creating an app on the Schwab Developer Portal; equivalent to OAuth's `client_id`. **Not technically a secret**, but in practice still must be kept private (whoever has it can initiate OAuth flows in your app's name). |
| **App Secret** | The ~16-character client credential issued by Schwab; equivalent to OAuth's `client_secret`. **Never** disclose under any circumstance; if leaked, immediately use the Portal's **Reset App Secret**. |
| **access_token** | The credential placed in `Authorization: Bearer <token>` for API calls; **valid for 90 minutes**. schwab-py auto-refreshes it before each API call. |
| **refresh_token** | The long-lived credential used to mint a new access_token; has a **hard 7-day lifetime**. After expiry you must re-run `login_flow` — there is **no way around it** (Schwab OAuth design). |
| **rotate-on-use** | Each refresh issues a new refresh_token and invalidates the old one immediately. This means token.json cannot be shared across machines. |
| **OAuth code** | The one-time ~6-character code that the browser is redirected back with during OAuth authorization; expires in 5 minutes; exchanged with the App Secret at the token endpoint for access_token + refresh_token. |
| **`login_flow`** | schwab-py's default OAuth mode; the CLI starts an https listener on `127.0.0.1:8182` and automatically captures the redirect. |
| **`manual_flow`** | Headless OAuth mode; the CLI prints the authorize URL and the user manually copies the redirect URL back. |
| **Callback URL** | The destination Schwab redirects to after OAuth; must be character-identical to `.env`'s `SCHWAB_CALLBACK_URL`. |
| **CSRF state** | A random string generated by schwab-py during OAuth; validated when the redirect comes back; protects against CSRF. |
| **TokenState** | The 4 enums returned by `health_check()`: `valid` / `missing` / `malformed` / `insecure_perms`. |

## MCP & skills

| Term | Definition |
| ---- | ---------- |
| **MCP** | Model Context Protocol, the LLM tool / resource protocol led by Anthropic. <https://modelcontextprotocol.io> |
| **MCP server** | A process that exposes tools / resources; in this project it refers to `schwab-marketdata-mcp`. |
| **MCP client** | A process that calls an MCP server; in this doc it refers to Cursor / Claude / your own Python/TS/Rust app. |
| **stdio transport** | The default MCP transport; client and server communicate via stdin/stdout using newline-delimited JSON-RPC. |
| **Skill** | The markdown-based knowledge package format for Cursor / Claude, containing a `SKILL.md` and a `references/` subtree. This repo provides two skills: `schwab-marketdata-ops-en` and `schwab-marketdata-workflows-en`. |
| **`compatible_mcp_version`** | A skill frontmatter field constraining the server-version range the skill is compatible with (semver-like). |
| **`required_workspace`** | A skill frontmatter field constraining the cwd subtree under which the skill is allowed to write data. |
| **`language_directive`** | A skill frontmatter field that forces the agent to respond in a specific language (this skill mandates English). |
| **Activation handshake** | The two-step check the agent must run when activating the skill: first call `get_server_info` to validate the version range, then call `health_check` to validate token state. |

## Schwab API & symbols

| Term | Definition |
| ---- | ---------- |
| **Market Data Production** | A Schwab Developer Portal API product that provides read-only market data: quotes / price history / option chain / etc. This server **only uses this product**; it does **not** call the Trader API. |
| **non-redistributable** | Schwab ToS explicitly states that Market Data is non-redistributable; any data fetched by this server must stay in private repositories / private storage. |
| **OSI option symbol** | The OCC Symbology Initiative's 21-character fixed format: `{ROOT:6}{YYMMDD:6}{C\|P:1}{STRIKE×1000:8}`. Pass it directly to `get_quote`. |
| **CUSIP** | The 9-digit US security identifier; accepted by `get_instrument_by_cusip`. |
| **ISIN** | The 12-digit international security identifier; for US securities ISIN = `US` + 9-digit CUSIP + 1 check digit. |
| **Cartesian product (price history)** | The `(period_type, period, frequency_type, frequency)` 4-tuple of `get_price_history` only accepts a restricted set of combinations; illegal combinations are **silently 400'd by the server**. |
| **Pydantic Literal enum names** | The public API always takes the enum name (`"VOLUME"`, `"NASDAQ"`, `"DJI"`); never pass schwab-py's wire value (`"$DJI"`, `"day"`). The MCP server translates internally. |
| **wire value** | The string schwab-py actually writes into the HTTP request; agents should not pass these directly. |

## Rate limiting & quota

| Term | Definition |
| ---- | ---------- |
| **token-bucket** | The local rate-limit implementation: bucket capacity = `SCHWAB_RATE_LIMIT_PER_MIN` (default 120); each request consumes 1 slot; slots auto-recover after 60 seconds. |
| **sliding window** | How token-bucket slots recover — not a per-minute reset, but each slot expiring 60 seconds after it was consumed. |
| **`SCHWAB_RATE_LIMIT_PER_MIN`** | Local bucket capacity ceiling, default 120; recommended to set stricter than Schwab's server-side limit (e.g. 100). |
| **`SCHWAB_MAX_RETRIES`** | Number of automatic retries on 429 / 5xx; default 2; set to 0 in tests / CI to surface failures immediately. |
| **rate_limit_warning** | When the bucket has < 20 slots remaining, the server emits a structured stderr log line: `{"event":"rate_limit_warning","remaining":N}`. |
| **`SchwabRateLimitError`** | The error dict returned by the server when the bucket is at 0 slots, or when Schwab returns 429. |

## Error system

| Term | Definition |
| ---- | ---------- |
| **`SchwabAuthError`** | OAuth / token failures; carries a `reason` field (one of 6 values: `refresh_token_expired` / `token_not_initialized` / `callback_url_mismatch` / etc.). |
| **`SchwabRateLimitError`** | Rate-limit failures; carries `retry_after_seconds` / `current_window_used` fields. |
| **`SchwabValidationError`** | Pydantic input validation failures; carries `field` pointing to the offending parameter. **Does not consume Schwab quota**. |
| **`SchwabTransientError`** | Schwab 5xx / network jitter; schwab-py already retried `SCHWAB_MAX_RETRIES` times unsuccessfully. |
| **error normalization** | All exceptions thrown by schwab-py are wrapped by the server into `{"error": "...", ...}` dicts and returned; they **do not** propagate as exceptions through the MCP protocol. |

## Workflows & outputs

| Term | Definition |
| ---- | ---------- |
| **Playbook** | A multi-step workflow defined in the `schwab-marketdata-workflows-en` skill (e.g. `voo-qqq-tracker-update.md`). Every playbook caps tool calls at ≤ 30. |
| **Pre-flight** | The mandatory 5-step check at playbook startup (version handshake / token health / cwd / repo privacy / idempotency). |
| **commit prefix `data(schwab):`** | All commits triggered by a playbook must use this prefix for audit traceability. |
| **`gh repo view --json isPrivate`** | Before any write, playbooks use the GitHub CLI to verify the target repo is private; **`isPrivate=false` aborts**. |
| **Idempotency window** | The time window during which a playbook re-run is idempotent (30 minutes); see the workflows skill. |

## File paths

| Term | Definition |
| ---- | ---------- |
| **token path** | `${XDG_STATE_HOME:-~/.local/state}/schwab-marketdata-mcp/token.json`; mode `600`, parent dir `700`. |
| **`SCHWAB_REAUTH_NEEDED.md`** | A marker file the health module writes to `~/Desktop/` when the token is about to expire or is corrupt. |
| **server.log** | The server's rolling log, default `${XDG_STATE_HOME}/schwab-marketdata-mcp/logs/server.log`, 10 MB × 5 rotated. |
| **`.env`** | At the MCP repo root; contains `SCHWAB_APP_KEY` / `SCHWAB_APP_SECRET` / `SCHWAB_CALLBACK_URL`; mode `600`; **never committed**. |

## Security

| Term | Definition |
| ---- | ---------- |
| **allow-list paths** | The server refuses tokens persisted outside `~/.local/state` / `~/.config` subtrees (`security.py`). |
| **cloud-sync detection** | The server detects whether the token landed in iCloud / Dropbox sync drives and raises `SchwabAuthError(reason="cloud_path_detected")`. |
| **`--i-understand-cloud-sync-risk`** | An auth-CLI override flag for cases where you knowingly want to bypass cloud-sync detection. |
| **`gitleaks` / `detect-secrets`** | pre-commit hooks that scan commits for secret patterns. |
| **Git Safety Protocol** | No force-push to main, no bypassing code review, no `--no-verify` to skip hooks. |

## Health probing

| Term | Definition |
| ---- | ---------- |
| **`schwab_marketdata_mcp.health`** | A standalone executable module (`uv run python -m schwab_marketdata_mcp.health`); prints health JSON and writes `SCHWAB_REAUTH_NEEDED.md` when needed. |
| **probe cadence** | Recommended: every 4 hours + Sundays 20:00 + Wednesdays 21:00. |
| **`notifier-self-test.sh`** | A script under the MCP repo's `scripts/` that self-tests the notification channel (macOS / Linux desktop notification + marker file). |
